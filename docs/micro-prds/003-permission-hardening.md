# Micro-PRD 003 — Permission Hardening

## Goal
Make all permission-sensitive flows App Review-safe and user-recoverable.

## Scope
- denied permission UX
- recovery links/instructions
- audit microphone/photos/camera/speech/Spotlight paths
- internal review checklist

## Acceptance Criteria
- permission denial states are explicit and recoverable
- no core feature silently fails when permission is denied
- App Review checklist exists in docs

## Complexity
Small–Medium
