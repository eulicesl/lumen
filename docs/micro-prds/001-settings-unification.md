# Micro-PRD 001 — Settings Unification

## Goal
Create one canonical settings architecture and UI for Lumen.

## Why
Lumen currently has duplicated settings surfaces, which makes the app feel unfinished and increases product drift.

## Scope
- audit all settings surfaces
- define final settings IA
- merge into one canonical settings entry point
- preserve model/privacy/memory/appearance access

## Acceptance Criteria
- only one primary settings surface remains
- all major settings categories are reachable from that surface
- no duplicated or contradictory settings flows remain

## Complexity
Medium
