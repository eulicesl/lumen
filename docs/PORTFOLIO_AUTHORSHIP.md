# Lumen Portfolio Authorship

Lumen is original product and engineering work by Eulices.
This codebase was built from scratch as a native SwiftUI application and maintained as a portfolio-grade product repository.

## What "original work" means here

- The repository is the primary source of truth for the app, not a forked application or a re-skinned template.
- Product scope, architecture, UI direction, feature selection, and release decisions are owned by Eulices.
- Code, docs, tests, and release workflow are developed directly in this repository under normal branch, review, and merge discipline.

## AI-assisted, human-owned workflow

Lumen uses AI the way a strong senior engineer would use modern tooling: to accelerate iteration, review changes, tighten documentation, and help execute well-scoped implementation tasks.

That does not change authorship.
Eulices remains responsible for:

- defining the product and roadmap
- deciding architecture and implementation direction
- reviewing and integrating changes
- setting the quality bar for code, UX, accessibility, and release readiness
- owning the shipped result

## Why this matters for portfolio review

This repository is intended to demonstrate senior-level engineering behavior, not just feature output.
Reviewers should expect to see:

- small, reviewable pull requests
- explicit validation and test evidence
- resolved review feedback
- documentation that stays aligned with shipped behavior
- release discipline suitable for a real App Store product
