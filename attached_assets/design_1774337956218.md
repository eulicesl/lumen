# Lumen AI — Design System

> iOS 26 Liquid Glass native design language. Minimal, clean, and privacy-focused.

**Version:** 1.0
**Last Updated:** March 20, 2026
**Platforms:** iOS 26, iPadOS 26, macOS 26

---

## 1. Design Philosophy

### Core Principles

1. **Let the glass speak** — Liquid Glass is our primary visual language. We don't fight it with heavy custom styling. Controls, navigation, and containers use native glass materials.
2. **Content over chrome** — The user's conversation IS the UI. Minimize decorative elements. Every pixel either shows content or helps the user act.
3. **Quiet confidence** — No flashy gradients, no mascots, no brand-heavy theming. The app feels like a natural extension of iOS 26.
4. **Progressive disclosure** — Simple by default. Advanced features reveal themselves when needed, never before.
5. **Accessible by design** — Not an afterthought. Color is never the only indicator. All text scales. All interactions have alternatives.

### Visual Identity

| Property | Value |
|----------|-------|
| Brand Color | System Tint (user-configurable, default: `.blue`) |
| Typography | San Francisco (system default) |
| Icons | SF Symbols 6 exclusively |
| Materials | Liquid Glass, system materials |
| Corners | Continuous corner radius (`.cornerRadius` with `.continuous`) |
| Motion | Spring animations matching system physics |

**Why no brand color?** Privacy apps shouldn't demand attention. Lumen blends with the user's system tint color, feeling like part of iOS rather than a third-party app.

### Semantic Colors (Light + Dark adaptive)

```swift
enum LumenColor {
    // MARK: - Text
    static let primaryText    = Color(.label)
    static let secondaryText  = Color(.secondaryLabel)

    static let primaryBackground    = Color(.systemBackground)
    static let secondaryBackground  = Color(.secondarySystemBackground)
    static let groupedBackground    = Color(.systemGroupedBackground)
    static let groupedSecondary     = Color(.secondarySystemGroupedBackground)

    static let userBubble      = Color.accentColor.opacity(0.15)
    static let assistantBubble = Color(.secondarySystemGroupedBackground)
    static let systemBubble    = Color(.tertiarySystemGroupedBackground)

    // MARK: - Status
    static let success  = Color.green
    static let warning  = Color.orange
    static let error    = Color.red
    static let info     = Color.blue

    // MARK: - Interactive
    static let tint          = Color.accentColor
    static let destructive   = Color.red
    static let disabled      = Color(.quaternaryLabel)

```

### Color Usage Rules

1. **Never hardcode hex colors** — Always use semantic system colors.
2. **Chat bubbles** — User messages get a tinted background. Assistant messages get secondary grouped background. Both adapt to dark mode automatically.
3. **Glass effects** — Use `.glassEffect()` on navigation elements, toolbars, and floating controls. Never on message bubbles (readability).
4. **High contrast mode** — Test all screens with Increase Contrast enabled. Borders and separators must be visible.

## 3. Typography

### Type Scale

All typography uses San Francisco via SwiftUI's built-in text styles. No custom fonts.

```swift
enum LumenType {
    // MARK: - Conversation
    static let messageBody    = Font.body           // 17pt - message text
    static let messageCaption = Font.caption         // 12pt - timestamps, metadata
    static let codeBlock      = Font.system(.body, design: .monospaced)

    // MARK: - Navigation
    static let largeTitle     = Font.largeTitle      // 34pt - screen titles
    static let title          = Font.title3          // 20pt - section headers
    static let headline       = Font.headline        // 17pt semibold - list items

    // MARK: - Controls
    static let buttonLabel    = Font.body            // 17pt - primary buttons
    static let footnote       = Font.footnote        // 13pt - secondary info
    static let caption2       = Font.caption2         // 11pt - badges, timestamps
}
```

### Typography Rules

1. **Dynamic Type required** — Every text element must use a text style, not a fixed point size.
2. **Monospaced for code** — All code blocks and inline code use `.monospaced` design.
3. **No bold abuse** — Only use `.headline` or `.bold()` for genuinely important labels. Over-bolding creates visual noise.
4. **Line spacing** — Chat messages use default line spacing. No custom `lineSpacing()` unless rendering long-form content.

    static let xxxl: CGFloat = 48    // Empty state vertical spacing

```

### Layout Grid

| Platform | Content Width | Edge Padding | Sidebar Width |
|----------|--------------|--------------|---------------|
| iPhone | Full width | 16pt | N/A (sheet) |
| iPad Portrait | Full width | 20pt | 320pt |
| iPad Landscape | Max 720pt centered | 20pt | 320pt |
| macOS | Max 800pt centered | 24pt | 280pt |

### Layout Rules

1. **Message bubbles** — Max width 85% of container on iPhone, 65% on iPad/Mac. Always left-aligned for assistant, right-aligned for user.
2. **Touch targets** — Minimum 44x44pt for all interactive elements (Apple HIG requirement).
3. **Safe areas** — Always respect safe areas. Never `ignoresSafeArea()` except for backgrounds.
4. **Keyboard avoidance** — Input bar pins above keyboard with spring animation. Messages scroll to maintain position.

## 6. Liquid Glass Usage Guide

### Where to Apply Glass

| Element | Glass | Rationale |
|---------|-------|-----------|
| Tab bar | Yes (automatic) | System behavior in iOS 26 |
| Navigation bar | Yes (automatic) | System behavior |
| Toolbar | Yes (automatic) | System behavior |
| Floating input bar | Yes (`.glassEffect()`) | Elevates above content |
| Model picker pill | Yes (`.glassEffect()`) | Interactive overlay |
| Action buttons on messages | Yes (`.glassEffect()`) | Floating controls |
| Message bubbles | **No** | Readability — text on glass is hard to read |
| Settings form | **No** | Standard grouped form style |
| Code blocks | **No** | Needs solid background for contrast |
| Empty states | **No** | Clean, simple centered layout |

### Glass Implementation

```swift
// Floating input bar
HStack { ... }
    .padding()
    .glassEffect(.regular.interactive, in: .capsule)

// Model selector pill
Menu { ... } label: {
    Label(model.name, systemImage: "cpu")
}
.glassEffect(.regular.interactive, in: .capsule)

// Tab bar minimization on scroll (automatic in iOS 26)
TabView {
    // Tabs automatically minimize when content scrolls
}
.tabViewStyle(.tabBarOnly)

// Scroll edge effect for message list
ScrollView {
    LazyVStack { ... }
}
.scrollEdgeEffectStyle(.soft, for: .top)
```

### Glass Don'ts

1. **Don't stack glass on glass** — Avoid glass containers inside glass containers. One layer only.
2. **Don't use glass for text backgrounds** — Refraction makes text hard to read, especially for accessibility.
3. **Don't force glass on older devices** — Use `@available` checks and fall back to `.regularMaterial`.
4. **Don't animate glass properties** — Glass transitions should be opacity-based, not property-based.

## 8. Animation & Motion

### Spring Physics

```swift
enum LumenAnimation {
    // Standard spring for UI transitions
    static let standard = Animation.spring(duration: 0.35, bounce: 0.15)

    // Snappy spring for button presses and toggles
    static let snappy = Animation.spring(duration: 0.25, bounce: 0.2)

    // Gentle spring for large layout changes
    static let gentle = Animation.spring(duration: 0.5, bounce: 0.1)

    // Quick fade for content appearing/disappearing
    static let fade = Animation.easeOut(duration: 0.15)
}
```

### Motion Guidelines

| Action | Animation | Duration |
|--------|-----------|----------|
| Send message | Input clears + scroll to bottom | 0.25s spring |
| New token streams in | No animation (performance) | Instant |
| Message appears | Opacity fade-in | 0.15s ease-out |
| Sidebar toggle | Spring slide | 0.35s spring |
| Button press | Scale to 0.95, spring back | 0.25s snappy |
| Sheet presentation | System default | System |
| Tab switch | System default | System |
| Delete conversation | Swipe + fade | System list animation |

### Reduce Motion Support

```swift
@Environment(\.accessibilityReduceMotion) var reduceMotion

// Always check
withAnimation(reduceMotion ? .none : LumenAnimation.standard) {
    // state change
}
```

## 10. Dark Mode

### Rules

1. **System automatic** — Respect `colorScheme` environment value. Never force a mode.
2. **True black backgrounds** — On OLED devices, use `.systemBackground` (true black in dark mode).
3. **Elevated surfaces** — Use `.secondarySystemBackground` for cards/bubbles in dark mode (slightly elevated from black).
4. **Code blocks** — Always dark background regardless of mode (convention from IDEs).
5. **Glass in dark mode** — Glass materials automatically adapt. No manual adjustments needed.

## 11. Accessibility Checklist

Every screen must pass:

- [ ] **VoiceOver:** All interactive elements have labels. All images have descriptions. Navigation order is logical.
- [ ] **Dynamic Type:** Text reflows at all sizes (xSmall through AX5). No truncation of essential content.
- [ ] **Reduce Motion:** All animations have `reduceMotion` alternatives. Streaming text still works.
- [ ] **Increase Contrast:** All text meets WCAG AA (4.5:1 for body, 3:1 for large text).
- [ ] **Color Blindness:** No information conveyed by color alone. Status always has icon + text.
- [ ] **Switch Control:** All features accessible via switch. Tab order is logical.
- [ ] **Bold Text:** System bold preference respected. No fixed font weights.
- [ ] **Reduce Transparency:** Glass effects degrade gracefully to solid backgrounds.

### Assistive Access Mode

For users with cognitive disabilities, provide a simplified mode:
- Large buttons
- Fewer options
- Clear labels
- No nested navigation

## 12. Platform Adaptations

### iPhone → iPad

| Aspect | iPhone | iPad |
|--------|--------|------|
| Navigation | Tab bar (5 tabs) | Sidebar + detail |
| Chat width | Full width | Centered, max 720pt |

| Keyboard | Full screen keyboard | Floating or split keyboard support |
| Multitasking | N/A | Slide Over, Split View, Stage Manager |

### iPhone/iPad → macOS

| Aspect | iOS | macOS |

| Navigation | Tab bar / sidebar | Sidebar always visible |

| Input | Touch, keyboard | Keyboard-primary, trackpad |
| Menus | Context menus | Menu bar + context menus |
| Shortcuts | Limited | Full keyboard shortcut set |
| Quick access | Widget | Menu bar extra + ⌃⌘K panel |
