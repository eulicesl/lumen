## User Stories

### US5.1 — Branched Conversations
> As a user, I want to explore different conversation paths without losing the original.

**Acceptance Criteria:**
- "Branch from here" action on any message
- Creates a new conversation branch with shared history up to that point
- Visual branch indicator showing parent conversation
- Branch tree view showing all branches of a conversation
- Switch between branches preserving scroll position
- Delete branch without affecting parent or siblings
- Merge branches: copy selected messages from branch back to parent

### US5.2 — Persistent Memory
> As a user, I want the AI to remember things about me across conversations.

**Acceptance Criteria:**
- Memory panel in settings shows all stored memories
- AI automatically extracts key facts during conversations (opt-in)
- User can manually add/edit/delete memories
- Memories injected as context in new conversations
- Memory categories: Preferences, Facts, Instructions
- Export memory as JSON/Markdown
- Clear all memory with confirmation
- Memory is local-only (never sent to servers)

### US5.3 — Agent Mode
> As a power user, I want the AI to perform multi-step tasks autonomously.

**Acceptance Criteria:**
- "Agent mode" toggle in conversation header
- AI can plan multi-step tasks, execute them, and report results
- Step-by-step progress display with expandable details
- Tool usage shown with icons and descriptions
- User approval required for sensitive actions (configurable)
- Agent can access: web search (future), file system (sandboxed), device APIs
- Cancel agent execution at any step
- Agent history saved with conversation

### US5.4 — Tool Calling
> As a developer, I want models to call tools and return structured output.

**Acceptance Criteria:**
- Foundation Models tool calling via `@Generable` pattern
- Ollama tool calling for supported models
- Built-in tools: Date/Time, Calculator, Unit Conversion
- Tool activity feedback shown in real-time during execution
- Tool results displayed inline with appropriate formatting
- Custom tool registration (developer feature, future)

### US5.5 — Apple Health Integration
> As a health-conscious user, I want to discuss my health data with AI privately.

**Acceptance Criteria:**
- Health permission request with clear explanation
- Read health metrics: steps, heart rate, sleep, workouts, nutrition
- AI can answer questions about health trends
- All processing on-device (Foundation Models preferred)
- No health data sent to Ollama or external servers
- Clear data access indicator when health data is being used
- Configurable: which health categories to share

### US5.6 — Reminders & Calendar Integration
> As a user, I want the AI to help me manage tasks and events.

**Acceptance Criteria:**
- AI can suggest creating reminders from conversation context
- Reminder creation with title, due date, priority
- Calendar event creation with title, date/time, location
- Inline confirmation before creating any item
- EventKit permissions handled gracefully
- Deep link to created reminder/event

### Built-in Tools

struct DateTimeTool: AgentTool {
    let name = "get_current_datetime"
    let description = "Get the current date, time, and timezone"
    let parameters: [ToolParameter] = []

    func execute(arguments: [String: Any]) async throws -> String {
        let formatter = DateFormatter()
        formatter.dateStyle = .full
        formatter.timeStyle = .long
        return formatter.string(from: Date())
    }
}

struct CalculatorTool: AgentTool {
    let name = "calculate"
    let description = "Evaluate a mathematical expression"
    let parameters: [ToolParameter] = [
        .init(name: "expression", type: .string, description: "Math expression to evaluate")
    ]

    func execute(arguments: [String: Any]) async throws -> String {
        guard let expr = arguments["expression"] as? String else { throw ToolError.missingParameter }
        let expression = NSExpression(format: expr)
        guard let result = expression.expressionValue(with: nil, context: nil) else {
            throw ToolError.evaluationFailed
        }
        return "\(result)"
    }
}

## Privacy Guarantees

This phase introduces sensitive data integrations. Privacy rules:

1. **Health data** is ONLY processed by Apple Foundation Models (on-device). Never sent to Ollama or any server. A visual indicator appears when health data is in context.
2. **Memory** is stored in local SwiftData only. Never synced. Never sent as telemetry.
3. **Reminders/Calendar** access requires explicit permission and per-action confirmation.
4. **Agent actions** require user approval for anything that modifies system state.
