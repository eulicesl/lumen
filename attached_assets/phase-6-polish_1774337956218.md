## 6.1 — Performance Optimization

### Targets

| Metric | Target | Measurement |
|--------|--------|-------------|
| Cold launch | < 800ms | XCTest Performance |
| Warm launch | < 200ms | XCTest Performance |
| UI response to tap | < 100ms | Instruments Time Profiler |
| Token streaming latency | < 100ms from receipt to display | Custom instrumentation |
| TTFT (Time to First Token) | < 200ms (Foundation Models), < 500ms (Ollama local) | Custom instrumentation |
| Memory (idle) | < 50MB | Instruments Allocations |
| Memory (chat, 1000 messages) | < 150MB | Instruments Allocations |
| Scroll performance | 60fps sustained | Core Animation Instruments |
| Battery impact | < 5% per hour active chat | Energy Diagnostics |

### Optimization Tasks

- [ ] **LazyVStack optimization** — Ensure message list uses proper `id` for diffing. Pre-compute message heights.
- [ ] **Throttler tuning** — Profile streaming throttle interval. Adjust from 100ms if needed per device.
- [ ] **SwiftData query optimization** — Use `#Predicate` efficiently. Avoid fetching full message content for list views (fetch title + preview only).
- [ ] **Image memory** — Downscale thumbnails in conversation list. Full resolution only in detail view.
- [ ] **Markdown rendering cache** — Cache rendered AttributedString for messages that are complete (won't change).
- [ ] **Code block syntax highlighting** — Lazy-load highlighting. Only highlight visible code blocks.
- [ ] **Prefetching** — Prefetch adjacent conversations when scrolling sidebar.
- [ ] **Memory leaks** — Profile with Instruments Leaks. Fix any retain cycles in closures.
- [ ] **Background task efficiency** — Minimize wake-ups. Batch Spotlight index updates.

### Platform-Specific Performance

**iPhone:**
- [ ] Thermal throttling: Reduce animation complexity when thermal state ≥ `.serious`
- [ ] Low Power Mode: Disable Live Activities, reduce Spotlight indexing frequency

**iPad:**
- [ ] Stage Manager: Ensure proper layout in all window sizes
- [ ] Split View: Profile memory with two apps sharing screen

**macOS:**
- [ ] Menu bar extra: Minimal memory footprint when idle
- [ ] Floating panel: Appears in < 100ms from keyboard shortcut

## 6.3 — Edge Cases & Error Recovery

### Network Errors

- [ ] Ollama server unreachable: Show clear status, retry button, recent conversations still available offline
- [ ] Ollama timeout during streaming: Preserve partial response, show error with retry
- [ ] Ollama returns error (400/500): Parse error message, show user-friendly explanation
- [ ] Network change during stream: Attempt reconnection, preserve partial response

### Data Integrity

- [ ] SwiftData migration: Test upgrade path from v1 schema
- [ ] Corrupted database: Detect and offer rebuild from backup
- [ ] Large conversations (1000+ messages): Performance remains acceptable
- [ ] Concurrent access: Multiple views reading same conversation
- [ ] App killed during write: Data consistent on next launch

### Input Edge Cases

- [ ] Empty message: Prevent sending, disable send button
- [ ] Very long message (10,000+ chars): Send without UI freeze
- [ ] Rapid send: Prevent double-send with debouncing
- [ ] Special characters: Unicode, emoji, RTL text render correctly
- [ ] Paste large text: Handle gracefully with truncation warning

### Model Edge Cases

- [ ] Model deleted from Ollama while in use: Graceful error, prompt to select new model
- [ ] Foundation Models unavailable (device too old): Clear messaging, Ollama-only mode
- [ ] Model returns empty response: Show "No response generated" message
- [ ] Model returns malformed markdown: Fallback to plain text rendering
