## User Stories

### US1.1 — First Launch Experience
> As a new user, I want to start chatting immediately so I can evaluate the app quickly.

**Acceptance Criteria:**
- On first launch, if Apple Foundation Models are available, the app is ready to chat with zero configuration
- If no provider is available, show a friendly onboarding screen explaining how to set up Ollama
- Default conversation is created automatically
- Input field is focused and ready for typing

### US1.2 — Send and Receive Messages
> As a user, I want to send a message and see the AI response stream in real-time.

**Acceptance Criteria:**
- Typing in the input field and pressing Send/Return creates a user message
- The AI response streams token-by-token with < 100ms UI latency
- A typing indicator shows while waiting for the first token
- The message list auto-scrolls to the latest message
- The send button becomes a stop button during generation
- Pressing stop cancels the stream and keeps partial response
- Error states show inline with a retry button

### US1.3 — Markdown & Code Rendering
> As a user, I want AI responses to render markdown beautifully with syntax-highlighted code.

**Acceptance Criteria:**
- Headers, bold, italic, lists, links render correctly
- Code blocks show language label, syntax highlighting, and copy button
- Inline code renders in monospace with subtle background
- Tables render as formatted tables (not raw markdown)
- LaTeX/math renders if present (stretch goal)
- Think/reasoning blocks (`<think>`) are collapsible

### US1.4 — Conversation Management
> As a user, I want to manage my conversation history.

**Acceptance Criteria:**
- Sidebar/list shows all conversations grouped by date (Today, Yesterday, Last 7 Days, Last 30 Days, Older)
- Tap a conversation to switch to it
- Swipe to delete a conversation (with confirmation for conversations with > 5 messages)
- Long-press/right-click context menu: Rename, Pin, Duplicate, Delete
- New conversation button in toolbar/header
- Conversations auto-title based on first exchange (via AI)
- Empty conversation state shows suggestions

### US1.5 — Model Selection
> As a user, I want to choose which AI model to use for each conversation.

**Acceptance Criteria:**
- Model picker in chat header shows current model
- Picker lists all available models from all configured providers
- Models grouped by provider (Ollama, Foundation Models)
- Selecting a model applies to the current conversation
- Model preference persists per conversation
- Unavailable models show disabled state with reason

### US1.6 — System Prompts
> As a power user, I want to set custom system prompts per conversation.

**Acceptance Criteria:**
- Conversation settings sheet has a system prompt text field
- System prompt is sent as the first message in every API call
- Default system prompt configurable in Settings
- System prompt indicator visible in chat header when active

### US1.7 — Settings: Provider Configuration
> As a user, I want to configure my AI providers.

**Acceptance Criteria:**
- Ollama section: Server URL field, connection test button, status indicator
- Foundation Models section: Availability indicator, model download status
- Default model picker (across all providers)
- Bearer token field for Ollama (optional, secure storage)

### US1.8 — Platform Adaptation
> As a user, I want the app to feel native on iPhone, iPad, and Mac.

**Acceptance Criteria:**
- **iPhone:** Full-screen chat with bottom tab bar. Conversation list in Chat tab sidebar.
- **iPad:** NavigationSplitView with sidebar (conversation list) and detail (chat).
- **macOS:** NavigationSplitView with sidebar. Toolbar with model picker and new chat. Keyboard shortcuts for common actions.

### ChatStore

```swift
@Observable
@MainActor
final class ChatStore {
    static let shared = ChatStore()

    // State
    var conversations: [Conversation] = []
    var selectedConversation: Conversation?
    var messages: [ChatMessage] = []
    var conversationState: ConversationState = .idle
    var inputText: String = ""

    // Metrics
    var timeToFirstToken: TimeInterval?

    // Dependencies
    private let aiService = AIService.shared
    private let dataService = DataService.shared
    private let throttler = Throttler(interval: 0.1)

    func send() async {
        guard !inputText.trimmingCharacters(in: .whitespacesAndNewlines).isEmpty else { return }

        let userMessage = ChatMessage(role: .user, content: inputText)
        messages.append(userMessage)
        inputText = ""

        // Persist user message
        await dataService.addMessage(userMessage, to: selectedConversation!)

        // Start generation
        conversationState = .generating
        let startTime = Date()
        var assistantContent = ""

        let assistantMessage = ChatMessage(role: .assistant, content: "")
        messages.append(assistantMessage)

        do {
            let stream = aiService.chat(messages: messages, model: currentModel, options: chatOptions)
            for try await token in stream {
                if timeToFirstToken == nil {
                    timeToFirstToken = Date().timeIntervalSince(startTime)
                }
                assistantContent += token.text
                throttler.throttle { [weak self] in
                    self?.messages[self!.messages.count - 1].content = assistantContent
                }
            }
            conversationState = .idle
        } catch {
            conversationState = .error(error)
        }

        // Persist assistant message
        await dataService.addMessage(messages.last!, to: selectedConversation!)
    }

    func stopGeneration() { ... }
    func loadConversations() async { ... }
    func selectConversation(_ conversation: Conversation) async { ... }
    func deleteConversation(_ conversation: Conversation) async { ... }
    func renameConversation(_ conversation: Conversation, to name: String) async { ... }
}
```

### Markdown Rendering Strategy

**Option A: Native AttributedString (Recommended for iOS 26)**
- Use `AttributedString(markdown:)` for basic formatting
- Custom code block renderer with syntax highlighting
- Pros: Native, fast, no dependencies
- Cons: Limited table support, no LaTeX

**Option B: MarkdownUI library**
- Third-party library with full markdown support
- Pros: Feature-complete, tables, custom themes
- Cons: External dependency, may lag on iOS 26

**Decision:** Start with Option A (native), add MarkdownUI as fallback for advanced features.
