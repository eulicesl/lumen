## 1. Vision & Goals

### Vision
Lumen is a **privacy-first AI assistant** that runs entirely on your own infrastructure. No data leaves your device unless you choose it. Built exclusively for iOS 26, iPadOS 26, and macOS 26, Lumen is the most native-feeling AI chat app on Apple platforms — leveraging Liquid Glass design, Apple Foundation Models, and deep system integration to deliver an experience that feels like it was made by Apple.

### Goals

1. **Privacy by default** — All AI inference happens on-device (Apple Foundation Models) or on your own server (Ollama). Zero telemetry. No cloud dependency.
2. **iOS 26 native excellence** — Not a web wrapper. Not a port. Every pixel follows Apple HIG and Liquid Glass design language.
3. **ChatGPT/Claude feature parity** — Match the core capabilities of leading AI apps while being private.
4. **Apple ecosystem depth** — Siri, Spotlight, Widgets, Live Activities, SharePlay, Shortcuts, Health — deeper than any competitor.
5. **Open & extensible** — Open-source, free, and designed for community contribution.

### Non-Goals

- Supporting iOS < 26 (clean break, no backward compatibility)
- Cloud-hosted proprietary AI (no OpenAI/Anthropic API keys at launch — may add as optional later)
- Android or web versions
- Building our own language model

## 2. User Personas

### Privacy-Conscious Professional
> "I need a capable AI assistant but I don't trust cloud providers with my data."
- Runs Ollama on a home server or Mac
- Needs chat, document analysis, code assistance
- Values encryption, local-only data, no telemetry

### Apple Ecosystem Power User
> "I want an AI that works with my iPhone the way Apple's own apps do."
- Uses Siri Shortcuts, Widgets, Focus modes, Apple Health
- Expects Liquid Glass design, haptics, smooth animations
- Wants AI in Spotlight, Lock Screen, Dynamic Island

### Developer / Tinkerer
> "I want to run different models and compare their outputs."
- Self-hosts multiple Ollama models (Llama, Mistral, Gemma, etc.)
- Needs model comparison, prompt templates, system prompt control
- Values open-source, configurability, API access

### Casual User
> "I just want a free AI chat that works on my iPhone."
- Uses Apple Foundation Models (built into device, no setup)
- Needs simple chat, voice input, image analysis
- Expects it to "just work" like iMessage

## 3. Product Principles

1. **Privacy is non-negotiable** — Data stays on-device or on user-controlled servers. Period.
2. **Native > Web** — If Apple provides a framework, use it. No web views for core features.
3. **Minimal but capable** — Clean UI with progressive disclosure. Simple by default, powerful when needed.
4. **Performance is UX** — Sub-100ms UI responses. Streaming tokens must feel instant. Cold launch < 1s.
5. **Accessibility is not optional** — VoiceOver, Dynamic Type, Reduce Motion, Assistive Access from day one.
6. **Convention over configuration** — Sensible defaults. Zero-config for Apple Foundation Models. Minimal config for Ollama.

## 4. Architecture Overview

### Tech Stack

| Layer | Technology |
|-------|------------|
| Language | Swift 6 (strict concurrency) |
| UI | SwiftUI (iOS 26 APIs, Liquid Glass) |
| Persistence | SwiftData |
| State | @Observable, Actor-based services |
| Networking | URLSession + async/await streams |
| AI Providers | Ollama API, Apple Foundation Models |
| Testing | Swift Testing framework + XCTest |
| CI/CD | Xcode Cloud |
| Minimum Target | iOS 26, iPadOS 26, macOS 26 |

### Architecture Diagram

```
┌─────────────────────────────────────────────────────┐
│                    UI Layer                          │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐            │
│  │  Views   │  │ Widgets │  │  Intents│            │
│  └────┬─────┘  └────┬────┘  └────┬────┘            │
│       │              │            │                  │
│  ┌────┴──────────────┴────────────┴────┐            │
│  │          Store Layer (@Observable)   │            │
│  │  ChatStore · ModelStore · AppStore   │            │
│  └────────────────┬────────────────────┘            │
│                   │                                  │
│  ┌────────────────┴────────────────────┐            │
│  │         Service Layer (Actors)       │            │
│  │  AIService · DataService · Voice     │            │
│  └────────────────┬────────────────────┘            │
│                   │                                  │
│  ┌────────────────┴────────────────────┐            │
│  │        Provider Layer (Protocols)    │            │
│  │  OllamaProvider · FoundationModels  │            │
│  └─────────────────────────────────────┘            │
│                                                      │
│  ┌─────────────────────────────────────┐            │
│  │        Data Layer (SwiftData)        │            │
│  │  Conversation · Message · Model      │            │
│  └─────────────────────────────────────┘            │
└─────────────────────────────────────────────────────┘
```

### Key Architecture Decisions

1. **Protocol-oriented AI providers** — `AIProvider` protocol with `OllamaProvider` and `FoundationModelsProvider` conformances. Easy to add providers later.
2. **Actor-based services** — Thread safety by design. No `DispatchQueue` gymnastics.
3. **Single source of truth** — SwiftData models are the truth. Stores observe and project.
4. **Streaming-first** — All AI responses stream via `AsyncSequence`. UI throttles at 100ms.
5. **Feature modules** — Each major feature is a self-contained module (views + store + service).

### Provider Protocol

```swift
protocol AIProvider: Sendable {
    var id: String { get }
    var displayName: String { get }
    var isAvailable: Bool { get async }

    func models() async throws -> [AIModel]
    func chat(_ messages: [ChatMessage], model: AIModel, options: ChatOptions) -> AsyncThrowingStream<ChatToken, Error>
    func generate(_ prompt: String, model: AIModel, options: GenerateOptions) -> AsyncThrowingStream<String, Error>
}
```

## 7. Competitive Analysis

### Where Lumen Wins

1. **Privacy** — Zero cloud dependency. ChatGPT/Claude require cloud accounts and send all data to servers.
2. **Free forever** — No subscription tiers. All features available to everyone.
3. **Model flexibility** — Run any Ollama model. ChatGPT locks you to GPT. Claude locks you to Claude.
4. **Apple-native depth** — Widgets, Live Activities, Siri, Spotlight, Shortcuts, Health, Reminders — deeper than any competitor.
5. **Model comparison** — Side-by-side comparison is unique. Neither ChatGPT nor Claude offers this.
6. **Prompt library** — Built-in template system with variable substitution. Competitors have nothing like this.
7. **Open source** — Fully auditable. Users can verify privacy claims.

### Where Competitors Win (and how we mitigate)

| Competitor Advantage | Mitigation |
|---------------------|------------|
| ChatGPT has better base model quality | Apple Foundation Models improving rapidly; Ollama runs Llama 3.3, DeepSeek, etc. |
| Claude has 200K context window | Ollama models improving; smart context management helps |
| ChatGPT has image generation (DALL-E) | Out of scope v1. Ollama + Stable Diffusion possible future feature |
| Both have web search | Foundation Models + on-device capabilities. Web search plugin possible later |
| Both have multi-modal understanding | Vision models in Ollama + Apple Vision framework |

## 9. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Apple Foundation Models limited capability | Medium | High | Ollama as primary provider; Foundation Models for quick/simple tasks |
| Ollama API breaking changes | Low | Medium | Protocol abstraction layer; version pinning |
| iOS 26 API instability (new release) | Medium | Medium | Test on every beta; graceful fallbacks |
| Liquid Glass performance on older devices | Medium | Low | Profile early; reduce glass effects on A15 and below |
| App Store rejection (AI content policy) | Low | High | No content generation claims; user brings their own model |
| Scope creep across phases | High | Medium | Strict phase gates; ship each phase independently |
